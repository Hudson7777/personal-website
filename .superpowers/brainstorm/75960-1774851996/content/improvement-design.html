改进方案设计

12项改进 · 分为6个任务组 · 面向长期可维护性

1 删除孤立文件 styles/theme.ts

问题：该文件定义了一套完全不同的颜色系统（Emerald Green），从未被任何文件 import，与 Tailwind 配置中的 Monet 主题完全割裂，是认知噪音。
操作：直接删除 apps/frontend/src/styles/theme.ts。无副作用。

2 合并三个重复页面 → CategoryPage 通用组件

问题：Travel / Photography / History 三个页面代码完全相同，只有 category 字符串不同。
方案：创建 pages/CategoryPage.tsx，接受 category、title、subtitle、description、emptyMessage props。
Travel/Photography/History 各保留一行：return <CategoryPage category="travel" title="Travel" .../>
App.tsx 不变，路由结构保持不动。

3 删除 LatestArticles，统一用 ArticleGrid

问题：LatestArticles 与 ArticleGrid + ArticleCard 功能完全重叠，前者是后者的手写复制版本。
方案：在 Home.tsx 中将 <LatestArticles> 替换为 <ArticleGrid>，然后删除 LatestArticles.tsx。
注意：ArticleCard 已有 cover image、badge、excerpt、readTime，视觉效果与 LatestArticles 一致。

4 修复类型安全：消除 as any，统一 BadgeVariant 类型

根本原因：Badge 的 variant 类型是字面量联合 'primary' | 'secondary' | 'success' | 'warning' | 'error'，但调用方用 Record<string, string> 映射后变成了 string，需要 as any 转义。
方案：
  1. 在 Badge.tsx 中 export type BadgeVariant = 'primary' | 'secondary' | 'success' | 'warning' | 'error'
  2. 将各处的 Record<string, string> 改为 Record<string, BadgeVariant>
  3. ArticleCard、LatestArticles（删除前）、ArticleDetail 中的 as any 全部消除
  4. AdminArticleEditor 中的 payload as any 改为正确的 CreateArticleInput / UpdateArticleInput 类型

5 添加 Toast 通知系统（Zustand + ToastContainer）

新增文件：
  • stores/toast.ts — Zustand store，管理 toast 队列（id, message, type, duration）
  • components/ToastContainer.tsx — 渲染 toast 列表，固定在右下角，带入场/出场动画
  • hooks/useToast.ts — 便捷 hook，暴露 toast.success()、toast.error()、toast.info()
集成：在 Layout.tsx 中挂载 <ToastContainer />。在 Admin 页面的保存/删除操作成功/失败时调用 useToast。
样式：Monet sage 主题，success=sage绿，error=lavender红，info=mist蓝。

6 添加 ErrorBoundary + 404 NotFound 页面

新增文件：
  • components/ErrorBoundary.tsx — React class component，捕获渲染错误，显示友好错误页面（Monet 风格）
  • pages/NotFound.tsx — 404 页面，带"回到首页"按钮
集成：
  • 在 App.tsx 的 <Router> 外层包裹 <ErrorBoundary>
  • 在 Routes 末尾添加 <Route path="*" element={<NotFound />} />
总计：新增 5 个文件，修改约 12 个文件，删除 2 个文件（theme.ts、LatestArticles.tsx）。
